# RемнаShop — веб-кабинет и админка

Дополнение поверх Telegram-бота [RемнаShop](https://remnashop.mintlify.app/)
(образ `ghcr.io/snoups/remnashop`). Добавляет:

- **Веб-кабинет** (React) — подписка, устройства, рефералы, поддержка, оплата;
  вход через Telegram или по email с подтверждением.
- **Расширенную админку/API** (FastAPI) — пользователи, подписки
  (выдать/продлить/удалить вручную), тарифы, шлюзы, промокоды, транзакции,
  рассылки, статистика, тикеты, состояние нод Remnawave в реальном времени.

Корневой `Dockerfile` накладывает `admin_src/src/` на базовый образ бота;
кабинет собирается отдельно (`cabinet/`, Vite → nginx).

## Установка

> Предполагается, что бот RемнаShop **уже настроен** и его `.env` заполнен.

```bash
git clone https://github.com/alexdsndr161rus2015-maker/remnashop-cabinet.git
cd remnashop-cabinet
./install.sh
```

Скрипт **не трогает** существующую конфигурацию бота — он только **дописывает
недостающие** переменные этого дополнения:

- секреты web-части (`APP_API_KEY`, `APP_JWT_SECRET`) — генерирует сам;
- спрашивает только то, что нельзя сгенерировать — **username бота** и
  **публичный URL кабинета**;
- email (`EMAIL_*`) — по желанию (нужно для регистрации по почте).

Затем собирает overlay-образ бота и кабинет и поднимает всё через Docker
Compose. Уже заданные переменные остаются как есть; повторный запуск безопасен.

Нужны установленные **Docker** и **Docker Compose**.

> Если ставите на тот же сервер, где уже лежит каталог бота — поместите файлы
> этого репозитория в каталог бота (рядом с его `.env` и `docker-compose.yml`)
> и запустите `./install.sh` оттуда.

## После установки

| Сервис   | Адрес              | Назначение                      |
|----------|--------------------|---------------------------------|
| Бот/API  | `127.0.0.1:5000`   | вебхуки Telegram, публичный API |
| Кабинет  | `127.0.0.1:5002`   | веб-кабинет (nginx)             |

Направьте домены на сервер и настройте reverse-proxy с TLS (Caddy/nginx)
на порты `5000` (API/вебхуки) и `5002` (кабинет).

Полезные команды:

```bash
DC="docker compose -f docker-compose.yml -f cabinet/docker-compose.cabinet.yml"
$DC ps
$DC logs -f
$DC up -d            # применить изменения .env
```

## Конфигурация

Все переменные (бота и дополнения) описаны в [`.env.example`](.env.example).
Раздел «Добавлено RемнаShop» — то, что дописывает `install.sh`.

> `.env` содержит секреты и **не коммитится** (см. `.gitignore`).
